AI 工具分享（算法研发向）

Trae IDE 常用功能与真实开发案例

面向研发的 1 小时分享设计：正式讲解 30 分钟 + 互动答疑 30 分钟。

文档入口： Trae IDE 文档。 配套参考： Trae个人经验分享。

1. 分享目标与时间安排 5分钟

  • 聚焦 Trae 中的关键能力并映射到算法研发流程。
  • 按“30 分钟主讲 + 30 分钟答疑”组织可落地内容。
  • 让团队拿走可复用的方法（模板、清单、流程）。

2. Trae IDE 重点功能汇总 15分钟

Solo 模式

  • 强调个人任务闭环，减少上下文切换。
  • 适合快速产出 baseline 与实验脚本。

Plan + Spec

  • 先拆解计划，再定义可执行规格。
  • 先定指标和验收标准，再写代码。

MCP + Skill

  • MCP：接入外部工具与数据源。
  • Skill：沉淀可复用工作流（如特征巡检）。

上下文压缩

  • 长链路任务保留关键事实与结论。
  • 多轮调参时提升对话连续性。

沙箱

  • 在受控环境执行脚本与命令验证。
  • 降低误改主环境和依赖污染风险。

智能体（Agent）

  • 适合多步骤任务自动推进。
  • 可串联“读代码→改代码→验证→总结”。

3. 案例1：二分类 baseline 4分钟

  • 用 Plan 拆解数据、训练、评估和交付。
  • 用 Spec 固化输入输出和指标约束。
  • 在 Solo 模式快速产出可运行脚本。

4. 案例2：训练异常排查 4分钟

  • 由 智能体 分步分析日志并定位 nan 根因。
  • 用 上下文压缩保留关键排障线索。
  • 在 沙箱 先验证修复方案再回主环境。

5. 案例3：推理性能优化 4分钟

  • 用 Spec 定义 P95 延迟与吞吐目标。
  • 用 Skill 复用 profile 与优化 checklist。
  • 通过 MCP 接入监控数据做效果回归。

6. 方法论与答疑引导 33分钟

  • Prompt 四段式：背景 / 上下文 / 约束 / 输出。
  • 先小后大：MVP → 边界条件 → 性能优化 → 单测文档。
  • Q&A 建议：Solo、Agent、Plan/Spec 与 MCP/Skill 如何组合落地。
