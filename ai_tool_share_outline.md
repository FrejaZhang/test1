## 2. Trae 常用产品功能介绍（15分钟）

### 2.1 对话式代码助手（Chat / Ask）
- 适用：快速生成函数、解释报错、补齐测试。
- 对算法工程师价值：
  - 快速写 baseline（如特征工程、损失函数、评估函数）。
  - 对复杂报错（shape mismatch、CUDA OOM、依赖冲突）做“定位-修复”建议。

### 2.2 仓库级上下文理解（Repo Context）
- 适用：理解现有项目结构、快速 onboarding。
- 价值：
  - 让 AI 基于当前代码库回答“这个训练入口在哪里”“推理链路怎么走”。
  - 减少“脱离上下文”的泛化回答。

### 2.3 代码编辑与重构（Edit / Refactor）
- 适用：批量改函数签名、模块拆分、命名规范统一。
- 价值：
  - 从 notebook 原型迁移到工程化目录（`dataset/`、`models/`、`trainer/`）。
  - 自动补 docstring、type hints、错误处理。

### 2.4 调试与问题定位（Debug Assistance）
- 适用：日志分析、异常复现、性能瓶颈定位。
- 价值：
  - 分析训练日志异常（loss nan、评估波动）。
  - 给出可执行的排查 checklist（数据→模型→优化器→学习率）。

### 2.5 单测与脚本生成（Test / Script Generation）
- 适用：自动生成 pytest、数据校验脚本、实验脚本。
- 价值：
  - 为核心算法模块补最小单测，降低重构风险。
  - 生成批量实验脚本（不同 seed、lr、batch size）。

### 2.6 文档与知识沉淀（Doc / README / Prompt 模板）
- 适用：自动整理实验结论、模块说明、交接文档。
- 价值：
  - 让“个人经验”变“团队资产”。
  - 固化可复用 prompt 模板（数据清洗、特征排查、错误修复）。

## 3. 结合算法开发的实际案例（12分钟）

### 案例1：二分类模型从 0 到 1（4分钟）
**场景**：快速搭建一个用户流失预测 baseline。
- AI 协助内容：
  1. 生成数据预处理 pipeline（缺失值、类别编码、标准化）。
  2. 生成训练脚本与评估指标（AUC、F1、PR 曲线）。
  3. 生成实验记录模板（参数、结果、结论）。
- 产出：可运行 baseline + 初版实验报告。

### 案例2：排查训练异常 loss=nan（4分钟）
**场景**：深度模型训练第3轮出现 nan。
- AI 协助内容：
  1. 分析日志并给出优先级排查路径。
  2. 自动插入梯度裁剪、输入校验、数值稳定处理（如 `eps`）。
  3. 给出最小复现实验脚本。
- 产出：可复现问题 + 修复验证闭环。

### 案例3：推理性能优化（4分钟）
**场景**：线上推理延迟过高。
- AI 协助内容：
  1. 建议 profile 方案（CPU/GPU 时间分解）。
  2. 给出优化方向（batching、ONNX、半精度、缓存策略）。
  3. 生成对比实验表格模板。
- 产出：优化路线图 + 验证清单。

## 4. 高效使用 Trae 的方法论（3分钟）

### 4.1 Prompt 结构（推荐模板）
- 背景：任务与业务目标。
- 上下文：相关文件、函数、错误日志。
- 约束：技术栈、性能目标、代码风格。
- 输出：期望格式（代码 / diff / 表格 / 结论）。

### 4.2 “先小后大”迭代策略
- 先让 AI 给最小可运行版本（MVP）。
- 再补边界条件、性能优化、可观测性。
- 最后补单测与文档。

### 4.3 质量保障清单
- 关键路径必须人工 review。
- 单测覆盖核心逻辑。
- 指标回归（离线+线上）通过后再合并。
